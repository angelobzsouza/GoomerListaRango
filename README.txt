Desafios encontrados
		- O maior desafio para desenvolver a aplicação foi determinar qual seria a melhor forma de armazenar os dados no banco, uma vez que a resposta dessa pergunta pode veriar bastante de acordo com modo como as informações serão usadas no fututo. Por não ter conhecimento do caso de uso real e completo, optei pelo caminho que faria com que eu desenvolvesse mais rápido, uma vez que tempo era um requisito real e atual da aplicação enquanto problemas com desempenho não seriam tão grandes neste momento.
		- Outros desafio foi decidir qual seria a maneira mais organizada de estruturar o código, uma vez que este item não é apenas um critério de avaliação, mas também impacta diretamente no meu desempenho ao longo de desenvolvimento.
		- Por fim, dois ultimos desafios de certa forma complementares foram, uma certa insegurança para desenvolver algo que não necessáriamente será utilizado, mas sim avaliado e o tempo, que por mais que tenha sido suficiente, acabou se tornando curto devido a fatores externos.

Maneiras de melhorar a aplicação
	Existem diversas maneira de melhorar a aplicação e irei listar algumas a baixo:
		- Trabalhar as validações dos campos, adequando-as aos casos de uso reais da API
		- Alterar a estrutura do banco. Está possibilidade somente se aplicaria caso, devido as buscas que serão feitas, outra modelagem se encaixe melhor.
		- Tratamento de imagens, criando miniaturas para servir ao usuário, definindo formatos, tamanhos e dimensões.

Instruções para rodar a aplicação
	As instruções para rodar a aplicação se encontram na documentação da API que se encontra no mesmo respositório deste arquivo. Para testa-la, é possível utilizar o software opensource Postman.
